# DBRE/DBA ハンズオン

## 📖 このハンズオンについて

本ハンズオンは、データベース経験の少ない方がDBREやDBAとして実務で活躍できるレベルまで到達することを目標としています。

**学習の方針:**
- **PostgreSQL** を徹底的に学び、RDBMS の核心概念を完全理解
- **MySQL** は PostgreSQL との差分のみ習得（効率的学習）
- **Redis/MongoDB** で NoSQL アーキテクチャを理解

## 🎯 推奨学習パス（3つのプラン）

### 📘 プラン1: 標準プラン（20週間）- 完全マスター

**対象**: 時間に余裕があり、DBREとして完璧に習得したい方

#### Phase 1: 基礎理論（2-3週間）
- [基礎理論](./basic_theory.md) - アルゴリズム、アーキテクチャ、ACID、ストレージエンジン

#### Phase 2: RDBMS 徹底マスター（6-8週間）
- [PostgreSQL](./postgresql.md) - **メインDB**: MVCC、インデックス、パフォーマンスチューニング、ログ分析
- [MySQL](./mysql.md) - **差分学習**: PostgreSQL との違い（2週間）

#### Phase 3: NoSQL 実践（各4-6週間）
- [Redis](./redis.md) - インメモリDB、キャッシング、分散ロック
- [MongoDB](./mongodb.md) - ドキュメントDB、シャーディング

#### Phase 4: 統合運用（継続学習）
- マルチDB環境の設計、監視、マイグレーション

---

### ⚡ プラン2: 集中プラン（8-10週間）- 実務優先

**対象**: 早く実務で使えるレベルに到達したい方

#### Week 1-2: 基礎理論（圧縮版）
- [基礎理論](./basic_theory.md) の Chapter 1-3 のみ
  - データベース階層構造
  - B-tree インデックス
  - ACID 実装メカニズム
- **スキップ**: 詳細な内部実装、高度な理論

#### Week 3-6: PostgreSQL（実践重視）
- [PostgreSQL](./postgresql.md) の以下に集中:
  - ✅ 第1章: MVCC アーキテクチャ
  - ✅ 第2章: テーブル設計とインデックス
  - ✅ 第4章: EXPLAIN 分析
  - ✅ 第5章: 負荷テスト
  - ✅ 第7章: ログ分析（基本のみ）
- **スキップ**: 拡張機能詳細、マイグレーション、高度な最適化

#### Week 7: MySQL（差分のみ）
- [MySQL](./mysql.md) の第1章のみ
  - InnoDB vs PostgreSQL
  - Clustered Index の違い
  - レプリケーション差分

#### Week 8-9: Redis or MongoDB（どちらか選択）
- **Webアプリ向け**: [Redis](./redis.md) - キャッシング、セッション管理
- **データ分析向け**: [MongoDB](./mongodb.md) - ドキュメント設計、アグリゲーション

#### Week 10: 実践課題
- 選択したDBの実践課題を解く
- 簡単な本番環境構築

---

### 🚀 プラン3: 最速プラン（4-6週間）- 基礎のみ

**対象**: とにかく早く基本を押さえたい方、特定のDBに特化したい方

#### Week 1: 基礎理論（最小限）
- [基礎理論](./basic_theory.md) Chapter 1-2
  - インデックスの仕組み（B-tree）
  - トランザクションの基礎

#### Week 2-4: PostgreSQL（核心のみ）
- 第1章: アーキテクチャ理解
- 第2章: 基本的なテーブル設計
- 第4章: EXPLAIN の読み方
- **ハンズオン**: 環境構築 → テーブル作成 → インデックス → EXPLAIN

#### Week 5: 差分学習
- **午前**: MySQL の違い（InnoDB、レプリケーション）
- **午後**: Redis または MongoDB の基礎

#### Week 6: 実践演習
- 実際のアプリケーションで使ってみる
- パフォーマンス問題を体験して解決

---

## ⏱️ さらに短縮する方法

### 既知スキップ戦略
- **SQL経験者**: 基礎理論スキップ → **-2週間**
- **PostgreSQL経験者**: Phase 2スキップ → **-6週間**
- **NoSQL経験者**: Phase 3スキップ → **-8週間**

### 目的別カスタマイズ（最短6週間）

| 目的 | 学習内容 | 期間 |
|------|---------|------|
| **Webアプリ開発** | PostgreSQL + Redis | 6週 |
| **データ分析** | PostgreSQL + MongoDB | 7週 |
| **インフラ運用** | PostgreSQL + MySQL + Redis | 9週 |

## 💡 効率的な学習のコツ

**✅ 推奨**
- ハンズオン必須（読むだけ×）
- 毎日30分 > 週末まとめて
- アウトプット（ブログ、勉強会）
- 実務で即実践

**❌ 非推奨**
- 完璧主義（80%理解で次へ）
- 順番飛ばし
- 並行学習（混乱の元）
